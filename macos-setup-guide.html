준비 · 터미널이 뭔가요?

~2분

터미널(Terminal)은 마우스 클릭 대신 명령어로 컴퓨터를 조작하는 검정 창입니다. 개발자가 거의 모든 셋업 작업을 여기서 합니다. 처음엔 무서워 보여도, 명령어 한 줄씩만 따라치면 됩니다.

1) Terminal.app 실행하기

맥에는 기본 터미널 앱이 처음부터 깔려있습니다. 세 가지 방법 중 아무거나:

  • Spotlight 검색 (가장 빠름) · ⌘ + Space 누르고 → "터미널" 또는 "terminal" 입력 → Enter
  • Launchpad · F4 키 → 폴더 "기타(Other)" → "터미널" 클릭
  • Finder · 응용 프로그램 → 유틸리티 → 터미널.app

2) 터미널 화면 이해하기

처음 열면 이런 형태의 글자가 보입니다:

terminalewonyu@MacBook ~ %

해석: 사용자명@컴퓨터이름 현재위치 % — % 뒤에 깜빡이는 커서가 명령어 입력 자리. ~ 표시는 "내 홈 폴더"를 의미.

3) 첫 명령어 시도

가볍게 손에 익혀봅시다. 아래를 한 줄씩 치고 Enter:

terminalpwd          # 현재 위치 출력 (Print Working Directory)
ls           # 현재 폴더의 파일 목록 (LiSt)
whoami       # 내 사용자명
date         # 현재 시각

# 뒤는 주석(설명)이라 안 쳐도 됩니다. 명령어만 치고 Enter.

💡 3가지만 기억하세요
  • 복사·붙여넣기 가능 — 가이드의 코드블록 복사해서 ⌘+V 하면 됩니다
  • Enter를 쳐야 실행 — 입력만 하고 멍 때리면 아무 일도 안 일어남
  • 비밀번호 입력 시 화면에 안 보임 — 별표(*)도 안 뜨지만 정상. 그냥 치고 Enter

4) 터미널 닫기 / 다시 열기

terminalexit         # 터미널 세션 종료

또는 그냥 창 닫기(⌘+W). 다음 작업 때 다시 Spotlight로 열면 됩니다.

📍 다음 단계부터 · 이 가이드의 모든 코드블록은 터미널에 붙여넣고 Enter치면 됩니다. 나중에 Step 3에서 더 좋은 터미널인 Ghostty로 갈아타지만, 그 전까지는 기본 Terminal.app으로 진행합니다.

zsh 셋업

~10분

macOS Catalina(2019) 이후로 zsh가 기본 셸입니다. 설치는 필요 없고, 환경 설정과 oh-my-zsh로 사용성을 끌어올립니다.

1) 현재 셸 확인

terminalecho $SHELL          # /bin/zsh 가 나오면 OK
zsh --version        # zsh 5.x.x
참고 · 만약 bash가 나오면(오래된 macOS) 다음 명령으로 zsh로 전환: 
chsh -s /bin/zsh

2) ~/.zshrc 기본 파일 만들기

홈 디렉토리에 zsh 설정 파일이 없으면 생성합니다.

terminaltouch ~/.zshrc

3) Oh My Zsh 설치 (강력 추천)

zsh 플러그인·테마·자동완성을 한방에 깔아주는 프레임워크. 거의 표준입니다. 

terminalsh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

설치되면 ~/.zshrc가 자동으로 백업되고 새 파일로 교체됩니다.

4) 추천 플러그인 추가

terminal# 자동완성 제안 (이전 명령어 기반)
git clone https://github.com/zsh-users/zsh-autosuggestions \
  ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

# 문법 하이라이팅
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git \
  ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

5) ~/.zshrc 편집

terminalnano ~/.zshrc       # 또는 'code ~/.zshrc' (VS Code 깐 후)

아래 줄을 찾아서 플러그인 추가:

~/.zshrcplugins=(
    git
    zsh-autosuggestions
    zsh-syntax-highlighting
)

# 테마 (기본은 robbyrussell, 더 화려한 거 원하면 agnoster, powerlevel10k)
ZSH_THEME="robbyrussell"

6) 적용

terminalsource ~/.zshrc
💡 팁 · 이제 명령어 일부만 입력해도 회색으로 자동완성 제안이 뜨고, 명령어가 유효하면 초록색·잘못되면 빨간색으로 색이 바뀝니다.

Homebrew

~5분

macOS 패키지 매니저. brew install <도구>로 거의 모든 개발도구 설치.

설치

terminal/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

PATH 등록

M-series에서 Homebrew는 /opt/homebrew에 깔립니다. zsh 환경이므로 ~/.zprofile에 등록: 

terminalecho 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
💡 왜 ~/.zprofile? · 로그인 시 1회 로드되는 파일이라 PATH 같은 환경변수 등록에 적합합니다. ~/.zshrc는 매 셸 세션마다 로드되어 명령어 alias·플러그인용으로 쓰입니다.

확인

terminalbrew --version       # Homebrew 4.x.x
brew doctor          # 환경 진단

Ghostty 터미널

~1분

Apple 출신 엔지니어가 만든 GPU 가속 터미널. Terminal.app 대체용으로 빠르고 가볍습니다. 

terminalbrew install --cask ghostty

설치 후 Spotlight(⌘+Space) → "Ghostty" → 이후 모든 작업은 Ghostty에서.

커스터마이즈 파일 위치: ~/.config/ghostty/config

Node.js

~3분

Claude Code, OMC, React 프로젝트 모두 Node 위에서 동작. 

terminalbrew install node

확인

terminalnode -v              # v20.x 이상
npm -v               # 11.x 이상
여러 Node 버전 관리 · 프로젝트별로 Node 버전을 바꿔야 할 일이 생기면 brew install fnm 또는 nvm 추가 설치 권장. 처음엔 brew node 하나로 충분합니다.

Git & GitHub 셋업

~5분

본인 정보 설정

terminalgit config --global user.name "Your Name"
git config --global user.email "you@example.com"

SSH 키 생성

terminalssh-keygen -t ed25519 -C "you@example.com"
# 엔터 3번 (기본 경로, 비밀번호 생략)

cat ~/.ssh/id_ed25519.pub | pbcopy
# → 클립보드에 공개키 복사됨

GitHub 접속 → Settings → SSH and GPG keys → New SSH key → 붙여넣기 → 저장.

연결 확인

terminalssh -T git@github.com
# Hi yourname! You've successfully authenticated...

gh CLI (선택, 추천)

terminalbrew install gh
gh auth login

Oh My Claude Code (OMC)

~2분

멀티 에이전트 오케스트레이션 레이어. 8단계 디자인 파이프라인, executor·planner·reviewer 같은 전문 에이전트, 자동 메모리 시스템을 제공.

Claude Code 실행한 상태에서:

claudesetup omc

또는 /oh-my-claudecode:omc-setup 슬래시 명령.

설치 확인

claude/oh-my-claudecode:omc-doctor

설치 결과는 ~/.claude/ 아래 agents·skills·hooks·memory 폴더로 깔립니다.

Figma MCP 연결

~5분

Figma 디자인을 Claude가 직접 읽고 React 코드로 변환할 수 있게 해줌.

1) Figma 데스크탑 앱 설치

terminalbrew install --cask figma

2) Figma Dev Mode MCP 활성화

Figma 데스크탑 → Preferences → Enable Dev Mode MCP Server 체크.

⚠️ 중요 · Figma는 반드시 데스크탑 앱이어야 합니다. 웹 브라우저 버전은 MCP 연결 안 됩니다.

3) Claude Code에서 MCP 등록

claude/oh-my-claudecode:mcp-setup

Figma 선택 → 인증 흐름 따라가기.

증상 해결
brew: command not found PATH 다시 등록: eval "$(/opt/homebrew/bin/brew shellenv)"
oh-my-zsh 색이 변함 없음 source ~/.zshrc 실행했는지, 새 터미널 창을 열었는지 확인

claude 인증 안 됨 브라우저 자동으로 안 열리면 출력된 URL을 직접 복사해서 열기
